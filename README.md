# Profile by form.

## one39files.co naming architecture

This project uses a delivery naming system based on deliverable subdomains and candidate slug paths:

- Candidate Profile: `profile.one39files.co/[candidate-slug]`
- Candidate Media Review: `media.one39files.co/[candidate-slug]`
- Candidate Review Package (future): `review.one39files.co/[candidate-slug]`

Candidate slugs are normalized from full names:

- lowercase
- whitespace + punctuation to hyphens
- diacritics removed
- collapsed hyphens
- no short aliases

Examples:

- `Jonathan Montgomery` -> `jonathan-montgomery`
- `James Nelson Jr.` -> `james-nelson-jr`
- `John David Smith III` -> `john-david-smith-iii`

## Output label standards

Public titles are standardized as:

- `Candidate Profile`
- `Candidate Media Review`
- `Candidate Review Package` (future)

## Routing behavior in app

The app maps template type to delivery channel when generating share links:

- `profile` and `profile_alt` templates → profile domain
- `media_review` template → media domain

Share links are generated by:

- `src/lib/delivery-routing.ts`
- used by:
  - `src/components/presentation/presentation-tab.tsx`
  - `src/routes/_authenticated/presentations.tsx`
  - `src/lib/email.functions.ts`

## Required environment variables

Set these for deployment:

- `APP_PROFILE_DOMAIN` = `https://profile.one39files.co`
- `APP_MEDIA_DOMAIN` = `https://media.one39files.co`
- `APP_REVIEW_DOMAIN` = `https://review.one39files.co`

Optional legacy fallback vars used in other flows:

- `APP_ORIGIN`
- `PUBLIC_APP_URL`

## Optional deployment model

You can either:

1) Keep existing path route and use subdomains per app host via rewrite/proxy
2) Route `/{shareSlug}` directly to the public share page and map that in DNS/CDN

A dedicated root slug route file exists at `src/routes/\$shareSlug.tsx`, which reuses:
- `src/routes/p.$shareSlug.tsx`

## Files for delivery system implementation

- `src/lib/presentation-templates.ts`
- `src/lib/delivery-routing.ts`
- `src/lib/presentations.functions.ts` (slug generation + persistence)
- `src/lib/email.functions.ts` (delivery URL + label updates)
- `src/components/presentation/presentation-tab.tsx` (copy share URL)
- `src/routes/_authenticated/presentations.tsx` (dashboard link display)
